# Plan 2 — What I Did

## What `_kg_build_final` / rdflib is — and when it was made

- **rdflib** is a Python library for RDF graphs. Loading triples into a `Graph` and
  *serialising* it produces **one clean document**: a single `@prefix` header, **no**
  `PREFIX` lines, **no** repeated headers.
  
- **`_kg_build_final`** is the function in `semRTI.py` that builds the final graph:
  read EXIF → run the CONSTRUCT templates (Section C/D) → read provenance (Section E) →
  **merge everything into a single rdflib graph** → write **one** file.

- **When was this made? — NOT during the T1–T12 work. It pre-dates my session.**
  `semRTI.py` is dated **22 June**; my session began **23 June**, and I never touched
  `semRTI.py`. So the "single clean document" mechanism already existed before I started.
  The old 4.4 MB `rupe-magna-final.ttl` came from a much older **cat-based concatenation**
  process — **T3/T4/T5 are that old file's problems.** For T3/T4/T5 I wrote nothing new;
  I only **verified** that the current pipeline's output is clean.
